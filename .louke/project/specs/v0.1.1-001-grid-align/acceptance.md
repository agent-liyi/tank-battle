## FR-0100 转向时网格对齐

### AC-1

AC-FR0100-01

- **给定** 玩家坦克位于 (x=100, y=68)，当前方向为右（水平）
- **当** 玩家按下方向键上（转向垂直方向）
- **那么** 坦克的 X 坐标 snap 到最近的网格线：Math.round(100/32)*32 = 96
- **并且** 坦克新位置为 (x=96, y=68)，随后开始向上移动

### AC-2

AC-FR0100-02

- **给定** 玩家坦克位于 (x=96, y=50)，当前方向为上（垂直）
- **当** 玩家按下方向键右（转向水平方向）
- **那么** 坦克的 Y 坐标 snap 到最近的网格线：Math.round(50/32)*32 = 64
- **并且** 坦克新位置为 (x=96, y=64)，随后开始向右移动

### AC-3

AC-FR0100-03

- **给定** 玩家坦克位于 (x=80, y=80)，当前方向为右
- **当** 玩家继续按下方向键右（方向未改变）
- **那么** 不触发网格对齐
- **并且** 坦克按正常速度水平移动，Y 坐标保持 80 不变

### AC-4

AC-FR0100-04

- **给定** 玩家坦克位于 (x=200, y=200)，处于静止状态（方向为 NONE）
- **当** 玩家按下方向键上
- **那么** 触发网格对齐（初始方向视为转向）
- **并且** 坦克 X 坐标 snap 到 Math.round(200/32)*32 = 192
- **并且** 坦克开始向上移动

### AC-5

AC-FR0100-05

- **给定** 玩家坦克位于 (x=65, y=130)，当前方向为下
- **当** 玩家按下方向键左（转向水平方向）
- **那么** 坦克的 Y 坐标 snap 到 Math.round(130/32)*32 = 128
- **并且** 坦克新位置为 (x=65, y=128)，随后开始向左移动

### AC-6

AC-FR0100-06

- **给定** 玩家坦克位于精确网格线上 (x=128, y=64)，当前方向为右
- **当** 玩家按下方向键上
- **那么** 坦克 X 坐标 snap 后仍为 128（已在网格线上）
- **并且** 坦克位置不变，开始向上移动

## FR-0200 对齐应用于玩家和敌方坦克

### AC-1

AC-FR0200-01

- **给定** 一辆敌方坦克位于 (x=100, y=50)，当前方向为下
- **当** AI 随机方向变更触发，新方向为左（水平）
- **那么** 敌方坦克的 Y 坐标 snap 到 Math.round(50/32)*32 = 64
- **并且** 敌方坦克新位置为 (x=100, y=64)，随后开始向左移动

### AC-2

AC-FR0200-02

- **给定** 一辆敌方坦克位于 (x=70, y=200)，当前方向为右
- **当** AI 随机方向变更触发，新方向为上（垂直）
- **那么** 敌方坦克的 X 坐标 snap 到 Math.round(70/32)*32 = 64
- **并且** 敌方坦克新位置为 (x=64, y=200)，随后开始向上移动

### AC-3

AC-FR0200-03

- **给定** 一辆敌方坦克位于 (x=50, y=50)，当前方向为下
- **当** 敌方坦克撞到边界，AI 选择新方向为右（水平）
- **那么** 敌方坦克的 Y 坐标 snap 到 Math.round(50/32)*32 = 64
- **并且** 敌方坦克在新方向上从对齐位置开始移动

## FR-0300 对齐边界安全

### AC-1

AC-FR0300-01

- **给定** 玩家坦克位于 (x=1005, y=68)，当前方向为右
- **当** 玩家按下方向键上（转向垂直方向）
- **那么** 坦克 X 坐标 snap 到 Math.round(1005/32)*32 = 992
- **并且** 992 + TANK_SIZE(30) = 1022 ≤ CANVAS_SIZE(1024)，在边界内
- **并且** 坦克开始向上移动

### AC-2

AC-FR0300-02

- **给定** snapToGrid 函数
- **当** 输入任意有效像素坐标
- **那么** 返回值始终为 TILE_SIZE(32) 的整数倍
- **并且** 返回值经 isWithinBounds 检查后始终在边界内（因最大对齐位置 992 + 30 = 1022 < 1024）

<a id="ac-fr-0400"></a>

## FR-0400 同向移动保持网格对齐

### AC-1

AC-FR0400-01

- **给定** 玩家坦克从网格对齐位置 (x=128, y=64) 开始向右移动
- **当** 坦克连续移动 8 帧（PLAYER_SPEED=4，8*4=32=TILE_SIZE）
- **那么** 坦克到达 (x=160, y=64)，Y 坐标仍为网格线 64
- **并且** 坦克始终在网格车道上

### AC-2

AC-FR0400-02

- **给定** 敌方坦克从网格对齐位置 (x=64, y=128) 开始向下移动
- **当** 坦克连续移动 10 帧（ENEMY_SPEED=3.2，10*3.2=32=TILE_SIZE）
- **那么** 坦克到达 (x=64, y=160)，X 坐标仍为网格线 64
- **并且** 坦克始终在网格车道上

## NFR-0010 对齐性能

### AC-1

AC-NFR0010-01

- **给定** snapToGrid 函数已实现
- **当** 在游戏循环中对每辆坦克（最多 5 辆：1 玩家 + 4 敌方）每帧调用一次
- **那么** 每次调用为 O(1) 的 Math.round + 乘法运算
- **并且** 不引入可测量的帧率下降（保持 60fps）

## NFR-0020 对齐可测试性

### AC-1

AC-NFR0020-01

- **给定** tank.js 模块导出 snapToGrid 函数
- **当** 执行 Jest 单元测试
- **那么** snapToGrid(0) === 0
- **并且** snapToGrid(15) === 0（向下取整到 0）
- **并且** snapToGrid(17) === 32（向上取整到 32）
- **并且** snapToGrid(32) === 32
- **并且** snapToGrid(100) === 96
- **并且** snapToGrid(130) === 128

### AC-2

AC-NFR0020-02

- **给定** 网格对齐逻辑已集成到 updatePlayer 函数
- **当** 使用确定性输入执行 Jest 集成测试（模拟方向从右变为上）
- **那么** 坦克 X 坐标在移动前被对齐到最近的网格线
- **并且** 测试可断言对齐后的精确坐标值

### AC-3

AC-NFR0020-03

- **给定** 网格对齐逻辑已集成到 updateEnemy 函数
- **当** 使用模拟 RNG 执行 Jest 集成测试（模拟敌方方向变更）
- **那么** 敌方坦克垂直于新方向的坐标被对齐到最近的网格线
- **并且** 测试可断言对齐后的精确坐标值
